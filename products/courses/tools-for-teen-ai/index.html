我见过太多人花大量时间研究ChatGPT的各种玩法，但就是做不出一个真正能用的产品。原因不是能力问题，而是工具选错了。

今天我要给你讲的5个工具，是我带过几十个学生、自己也在用的组合。这不是网上那种"必备100个AI工具"的水文，是真正帮你从零做出第一个产品的最小工具集。

不管你现在12岁还是22岁，学完今天这5个工具，你就具备了做出第一个AI产品的全部武器。Let's go。

为什么是这5个，不是别的？

不是"功能最全"，而是"从想法到产品最短路径"。每一个工具都在这条路上。

🔗 工具之间有链路

这5个工具不是孤立的，它们首尾相扣：写代码→托管→版本管理→部署，缺一不可。

✅ 学生真实验证

我带的学生用这套组合，最快3周就上线了第一个产品，不需要任何服务器知识。

在讲每个工具之前，我想先说一下我的选工具逻辑，这很重要。

市面上AI工具太多了，随便一搜就是"100个必备AI工具"。但我选工具只看一个标准：从想法到上线产品，这条路上哪个环节最卡，就用什么工具补哪个环节。

第二点，我要强调的是工具之间的链路。今天这5个工具不是孤立的，它们是一条完整的生产线：用AI写代码，在浏览器里直接运行，用Git管理版本，最后一键部署到全球都能访问的网址。这条链路缺任何一环都跑不通。

最后，学生验证这一点我要重点讲。我带过的学生，用这套工具组合，最快3周就上线了第一个产品。不需要买服务器，不需要学Linux，甚至不需要完全懂每行代码是什么意思。

好，我们开始看这5个工具。

// 工具 01 / 05

Claude Code
你的AI编程搭档

是什么

Anthropic出品的AI编程CLI工具。你用中文描述需求，它直接写代码、改代码、解释代码。

做什么

写整个项目的代码。从数据库结构到前端页面，从API接口到部署脚本，全部可以口述生成。

适合谁

所有人。零基础可以直接说"帮我做一个记单词的App"，有编程基础的用它加速10倍。

核心优势

可以直接读写本地文件、跑命令、调试错误——不是聊天，是真正在帮你干活。

claude.ai/code

第一个工具，Claude Code。这是我们今天整套流程的核心，没有它其他工具都跑不起来。

Claude Code是Anthropic——就是做Claude这家公司——出的一个命令行AI工具。它跟ChatGPT最大的区别是：它不只是聊天，它可以直接操作你电脑上的文件。你说"帮我在这个项目里加一个用户登录功能"，它直接改代码文件，你不需要复制粘贴。

适合谁？我要说：所有人。我带过完全零基础的初中生，第一次用Claude Code，3小时就做出了一个可以在浏览器里运行的小游戏。有编程基础的同学用它就更厉害了，原来要写一整天的功能，现在可能一小时就搞定。

记住它的核心定位：不是帮你学编程的老师，是帮你做产品的同事。这个定位很重要，会影响你怎么跟它交流。

// 工具 02 / 05

Cursor
AI驱动的代码编辑器

是什么

基于VS Code改造的AI编辑器。写代码时AI实时补全，Tab键接受建议，速度快到像"被AI附身"。

做什么

写代码时的主战场。浏览、修改、理解项目文件，AI会帮你补全下一行、解释报错、建议重构。

适合谁

有一点编程基础或者正在学编程的学生。完全零基础优先用Claude Code口述，有基础就切到Cursor。

跟Claude Code的关系

两者搭配使用。Claude Code负责"大改"，Cursor负责"细改"和日常写代码。

cursor.com

第二个工具，Cursor。这是我日常写代码用得最多的工具之一。

Cursor本质上是VS Code——就是微软那个最流行的代码编辑器——的AI增强版。它最厉害的地方是：你写代码的时候，AI会预测你下一步要写什么，按Tab就接受。我第一次用的时候感觉像是被AI附身，手速快了三倍。

有同学会问：Cursor和Claude Code有什么区别，要选哪个？答案是：两个都要用，搭配着来。Claude Code是"大改"——你想加一个新功能、重构一个模块，用Claude Code口述。Cursor是"细改"——你在写代码过程中需要补全、理解某段逻辑、修一个bug，用Cursor。

对完全零基础的同学，我的建议是先从Claude Code开始，做出第一个能跑的东西，再来学Cursor。两个工具同时上手容易混乱。

// 工具 03 / 05

Replit
浏览器里的开发环境

是什么

在线的代码运行环境，打开浏览器就能写代码、跑程序，不需要装任何东西。

做什么

快速验证想法、分享代码给朋友或老师看、做比赛项目演示——零配置，秒启动。

适合谁

初中生/高中生起步阶段最爱。电脑不够好、不想装一堆软件、需要随时随地写代码的人。

隐藏优势

Replit本身也有AI助手，做学校作业和小项目够用。内置分享链接，做完直接发链接给任何人看。

replit.com

第三个工具，Replit。这个工具对学生来说特别友好，我要重点介绍一下。

Replit最大的价值是零门槛。你不需要装Python，不需要配置Node.js环境，不需要学什么是终端——打开浏览器，登录，新建一个项目，直接开始写代码，立刻就能跑。

我遇到过很多学生，在自己电脑上装开发环境就卡了两三天，最后放弃了。这完全没必要。用Replit先做出东西，建立信心，再说本地环境的事。

还有一个场景我必须说：比赛演示。你做了一个AI项目，要在评委面前展示，用Replit做的项目直接发一个链接，评委打开就能用，不需要你带电脑去现场。这个优势在竞赛场景非常实用。

简单来说：想快速验证一个想法，用Replit。想做正式项目，再切到本地+Claude Code+Cursor。

// 工具 04 / 05

GitHub
你的作品集，不只是代码仓库

是什么

全球最大的代码托管平台，也是程序员的"简历"。你的所有项目代码都存在这里。

做什么

保存代码的每一次修改历史，防止改坏了找不回来；开源展示项目给招生官、面试官、投资人看。

申请加分

名校招生官真的会看GitHub。一个有50+ Stars的项目，比大多数竞赛证书含金量高得多。

现在就要建

越早开始，你的commit历史（贡献记录）越长。第一个项目不完美没关系，先有比完美更重要。

github.com

第四个工具，GitHub。这个工具跟前三个不一样，它不是帮你做东西更快，而是帮你把做好的东西变成资产。

先说一个很多家长不知道的事实：美国名校的招生官，真的会搜申请人的GitHub主页。这不是我猜的，是我亲眼见过的。我有一个学生，在GitHub上有一个AI项目拿了200个Star，他的招生文书就围绕这个项目写，最后进了MIT。

GitHub的第二个价值是代码历史。用Git管理代码之后，你改了什么、什么时候改的，全部有记录。改坏了？回滚。想看一个月前的版本？随时看。这是专业程序员的标准工作方式。

我对所有学生说的话：今天回去就注册GitHub，把你现在有的任何一个项目放上去。不完美没关系，先有再好。你的commit记录是有时间戳的，越早开始，到申请大学的时候，你的"开发历史"就越长越好看。

// 工具 05 / 05

Vercel
一行命令，全球上线

是什么

免费的网站部署平台。把你的GitHub代码连上Vercel，每次push代码就自动更新上线。

做什么

把你做的产品变成一个真实网址，任何人打开浏览器都能用。不需要买服务器，免费套餐够用。

为什么重要

"我做了一个AI App"和"这是我App的网址：xxx.vercel.app"——后者才是真正的产品。

跟GitHub的关系

GitHub存代码，Vercel运行代码。连接一次，之后每次GitHub更新，Vercel自动重新部署，全自动。

vercel.com

第五个工具，Vercel。这是整条链路的最后一环，也是最让人有成就感的一环。

我经常跟学生说一句话："做了但没上线，等于没做"。一个只能在自己电脑上跑的程序，是没有办法给别人用的，也没有办法拿去申请、参赛、展示。Vercel解决的就是这个问题——把你的代码变成一个真实的网址。

用法非常简单：把GitHub账号连接到Vercel，选择你要部署的项目，点一下Deploy，30秒后你就有一个`xxx.vercel.app`的网址了。全球任何地方的人，打开这个链接都能用你的产品。免费。

更厉害的是：之后你在Claude Code里改了代码、push到GitHub，Vercel自动检测到更新，自动重新部署，通常3分钟之内新版本就上线了。这就是专业团队的工作流，你今天就可以用。
